# Empirical Validation
**"איך נדע שזה עובד?"**

## העיקרון הבסיסי

> **לפני שאני (הסוכן) מסיים משימה, אנחנו ביחד צריכים לראות שזה עובד בפועל.**

"הקוד נראה נכון" ≠ אימות

## הפרוטוקול השיחתי

### 1️⃣ לפני שאני אומר "סיימתי"
אני שואל אותך:
- **"איך נדע שזה עובד?"**
- **"מה אתה רוצה לראות?"**

### 2️⃣ בוחרים ביחד מה לבדוק

| מה שינינו | איך נבדוק |
|-----------|-----------|
| מסך/ממשק | צילום מסך של התוצאה |
| פונקציה/קוד | הרצה והצגת הפלט |
| קובץ | בדיקה שנוצר/עודכן נכון |
| בניה/תצורה | build מוצלח |

### 3️⃣ אני מבצע ומראה לך
אני מריץ את הבדיקה ומציג **תוצאות אמיתיות** (לא "אמור לעבוד").

אם צריך, אני מתעד את ההוכחה ב-`CHANGELOG.md`.

### 4️⃣ אתה מאשר
- ✅ **"נראה טוב"** → המשימה הושלמה
- ❌ **"משהו לא נכון"** → חוזרים לתיקון

## ❌ ביטויים אסורים

אסור לי להשתמש באלה כהצדקה לסיום:
- "הקוד נראה נכון"
- "זה אמור לעבוד"
- "עשיתי דברים דומים בעבר"
- "זה עוקב אחרי התבנית"

## ✅ מה חובה להראות

**תוצאות אמיתיות** מהרצה, בדיקה, או צילום מסך.

## טיפול בכשלון

אם הבדיקה נכשלת:
1. ❌ **לא** מסמן את המשימה כהושלמה
2. 📝 מתעד את הכשל ב-`STATE.md`
3. 🔧 יוצר משימת תיקון אם הגורם ידוע
4. 🔍 אם 3+ כשלונות → מפעיל Context Health Monitor

## אינטגרציה

משתלב עם:
- `/verify` - הזרימה הראשית שמשתמשת בסקיל הזה
- `/execute` - חייב לאמת לפני סיום משימות
