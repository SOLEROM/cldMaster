---
name: specifications
description: Creates clear, professional specifications for projects and tasks. For new projects, runs a full specification process with deep research and vision clarification. For ongoing projects, breaks down tasks into milestones, tasks, and phases. Produces spec.md files as the project's source of truth.
---

In this way, the record, map and vision are all recorded and can be implemented in the right order, with the ability to come back to the "core specs" every time.

##Process for a new project:
1. Investigation - you and the architect should do a deep research and make the vision as clear as possible. you have the ability to use tools like search the web and use mcp servers when needed. the architect should answer questions to create the path as clear as possible. 
2. if the architect already created a specification plan, you should deeply review it and conclude:
- how can it be technically operated
- is the "stack" available for it is the most updated one - use context7 mcp for that if needed
- is the vision is something that can be realy done or is it a far fantasy..
3. If the architect has no specification plan start to build one with him. you must lead this part, because you are the professional advisor. Questions to ask for example, but you have to be free to go deeper if needed, and keep in mind that not all quiestions are needed for each project! You have to be aware of it..
Questioning example:
- What is the main target of your project/task.. for example: create a new financial folowup app to follow my incomes and outcomes. 
- Who will use this app 
- Do you need it to be secured (local for example) or its ok if the agent will see data like csv files with you business information (because if not, he should share only the titles line with the first content line of a csv file, to avoid it as much as possible)
- do you have any record in this kind of projects?
- How can you describe a progress with your project? for example - an excel file to be loaded and show a dashboard
- What is it you want to see there
- Is there any design specifications
- ETC....

4. The specification should be a 'SPEC.md' file in the .digido/ folder. 

5. Create "Milestones" inside this "SPEC.md" file. It should be kind of of main map to the project and should be easy to navigate.
