# Skill-Advicing workflow

## שלב 1: טען את הסקיל
קרא את הקובץ `.agent\skills\skill-advisor\SKILL.md`

## שלב 2: הבן את צרכיו של הארכיטקט
חקור אותו כדי להבין את הצרכים/החזון/הרצון, כדי להתאים את הskill הרלוונטי או ה-workflows הרלוונטיים.

לא למהר בבחירתם! להבין צרכים, לאחר מכן להציע אפשרויות, עם הסבר ברור מה כל skill/workflow עושה.

## שלב 3: הצעת השלב הבא
אחרי שתהליך היעוץ הסתיים, הצע לארכיטקט 'מה השלב הבא' בהתאם לתובנות השיחה בניכם.
