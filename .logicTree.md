# logicTree.md

## Structure Principles
- max_depth: 4 (user-approved exception from default 3)
- naming: snake_case
- index folders preferred
- placeholders must include "TBD"

## Top-Level Buckets

### setup/
- purpose: Getting Claude Code installed and configured
- contains: install/, config/, claude_md/, rules/, workspace/

### core/
- purpose: Core concepts, operating modes, models, and capabilities
- contains: cli/, concepts/, modes/, models/, superpowers/, permissions/, authority/

### extend/
- purpose: Extending Claude Code — hooks, plugins, skills, MCP, SDK, actions
- contains: hooks/, plugins/, skills/, mcp/, sdk/, actions/
- notes: hooks/how_to_guide/ contains non-markdown scripts (preserved in place)

### agents/
- purpose: Multi-agent patterns and task management
- contains: subagents/, parallel/, teams/, tasks/

### usage/
- purpose: Day-to-day operation — workflows, sessions, tips, optimization
- contains: workflow/, session/, checkpoints/, work_tree/, tips/, token_optimization/

### learn/
- purpose: Learning resources, evaluation, design patterns, hands-on labs
- contains: resources/, eval/, design/, playground/
- notes: playground/lab1/uige/ and playground/lab2_hooks/ are non-KB code projects

## Grouping Rules
- group by topic similarity
- if folder > 10 items → split
- avoid single-file folders unless they are clear conceptual/topic nodes
- prefer flat structure within each bucket (bucket/topic/file)

## Anti-Patterns
- deep nesting (> 4)
- camelCase names (use snake_case)
- duplicate folders for same topic (e.g. config + configs → merge)
- ungrouped large folders
