# Debug Logger

<role>
כשאתה כותב קוד Python, אתה **תמיד** מוסיף הדפסות (print statements) חכמות שמראות לארכיטקט מה קורה.

למה זה קריטי? כי הארכיטקט לא קורא קוד. הוא צריך **לראות** מה קורה בזמן ריצה - האם זה עובד, איפה זה תקוע, מה השתבש.

הקוד צריך לספר את הסיפור שלו - מה ניסה, מה קרה, האם הצליח.
</role>

---

## פורמט הדפסה

### מבנה בסיסי

```
[אימוג'י קטגוריה] הודעה קריאה עם פרטים ספציפיים
```

### 4 האימוג'ים

| אימוג'י | משמעות | מתי להשתמש |
|---------|---------|-----------|
| `[✅]` | הצלחה | הפעולה הושלמה בהצלחה |
| `[❌]` | שגיאה | הפעולה נכשלה |
| `[⚠️]` | אזהרה / ניסיון חוזר | משהו לא צפוי, אבל ממשיכים |
| `[ℹ️]` | מידע | מידע ניטרלי, הקשר, מצב |

### קטגוריות נפוצות

| קטגוריה | תחום |
|----------|------|
| `API` | קריאות לשרתים חיצוניים (OpenAI, Google, וכו') |
| `DB` | פעולות על מסד נתונים |
| `FILE` | קריאה/כתיבה של קבצים |
| `CONFIG` | טעינת הגדרות |
| `SETUP` | אתחול המערכת |
| `FLOW` | זרימת התהליך הכללית |

אפשר להמציא קטגוריות נוספות - רק לשמור על אותיות גדולות ועקביות.

## דוגמאות

```python
# כניסה לפונקציה
print(f"[ℹ️ API] קורא נתונים מ-OpenAI — מודל: gpt-4")

# הצלחה
print(f"[✅ API] התקבלה תשובה — {len(response)} תווים")

# שגיאה עם הקשר
print(f"[❌ API] נכשל — status: 429, הודעה: {error_msg}")

# ניסיון חוזר עם מונה
print(f"[⚠️ RETRY 2/3] ניסיון שני לחיבור לשרת...")

# פעולה על קובץ
print(f"[✅ FILE] נשמר בהצלחה — קובץ: output.pdf, גודל: 1.2MB")
```

## 5 הכללים החשובים

### 1. כל פונקציה מקבלת לוג

**מינימום:** כניסה + יציאה

```python
def fetch_user(user_id):
    print(f"[ℹ️ API] מביא משתמש — ID: {user_id}")

    try:
        result = api.get(f"/users/{user_id}")
        print(f"[✅ API] משתמש נמצא — שם: {result['name']}")
        return result
    except Exception as e:
        print(f"[❌ API] נכשל — {e}")
        raise
```

### 2. כל קריאת API - לפני **ואחרי**

```python
print(f"[ℹ️ API] שולח בקשה ל-OpenAI...")
response = openai.chat.completions.create(...)
print(f"[✅ API] התקבלה תשובה — tokens: {response.usage.total_tokens}")
```

**למה לפני ואחרי?** כי אם ה"אחרי" לא מופיע - אתה יודע בדיוק איפה זה תקוע.

### 3. ניסיונות חוזרים עם מונה

```python
for attempt in range(1, 4):  # 3 ניסיונות
    print(f"[⚠️ RETRY {attempt}/3] מנסה להתחבר...")
    try:
        connect()
        print(f"[✅ DB] התחבר בניסיון {attempt}")
        break
    except:
        if attempt == 3:
            print(f"[❌ DB] כל 3 הניסיונות נכשלו")
            raise
```

### 4. שגיאות עם הקשר מלא

```python
# ❌ רע - חסר הקשר
print(f"[❌] שגיאה: {e}")

# ✅ טוב - עם כל המידע
print(f"[❌ DB] נכשל להוסיף רשומה — טבלה: users, נתונים: {data}, שגיאה: {e}")
```

### 5. לעולם לא להדפיס סודות

**אסור בלוגים:** סיסמאות, API keys, טוקנים, מספרי אשראי

```python
# ✅ טוב - מסתיר חלק מה-token
print(f"[ℹ️ AUTH] משתמש ב-API key: {api_key[:4]}...{api_key[-4:]}")
```

## טעויות נפוצות להימנע מהן

### ❌ הדפסה בתוך לולאות

```python
# רע - 10,000 שורות של רעש
for item in items:
    print(f"[ℹ️] מעבד פריט {item.id}")
```

**תיקון:** הדפס סיכום

```python
print(f"[ℹ️ FLOW] מעבד {len(items)} פריטים...")
# ... עיבוד ...
print(f"[✅ FLOW] סיים לעבד {len(items)} פריטים")
```
